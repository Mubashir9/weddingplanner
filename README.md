# Wedding Planner

A shared planning tool built for me and my wife for our wedding.

---

## What it does

Five pages, each solving one problem:

**Dashboard** — the first thing you see. A countdown to the Shaadi, four stat cards (tasks done, budget spent, vendor count, venues confirmed), a list of your next upcoming tasks, and an AI nudge panel that reads your actual data and tells you what needs attention. The nudges are generated by Gemini 2.5 Flash and cached for 6 hours so they do not re-run on every load.

**Tasks** — a checklist grouped under each of the three events. Click a task's status badge to cycle it through `todo → in progress → done`. Add tasks inline per event. Both users see the same list in real time because it is all the same database.

**Vendors** — a card board for every vendor you are considering or have booked. Stores name, category, event, Instagram link, phone, quote in PKR, status, and notes. Filter by event or category. Cycle the status from shortlisted to contacted to booked with a single click.

**Budget** — line items grouped by event, plus a general bucket for costs that span all three. Each item has a paid toggle. Running total at the top. No budget ceiling enforced — add what you spend and see where you are.

**AI Chat** — a streaming conversation with Gemini that has your full wedding context loaded into every message. It knows your events, tasks, vendors, and budget. Useful for questions like "what should we focus on this week" or "how does our budget compare to a typical Shaadi in Lahore." It does not browse the internet.

### Key architectural decisions

**Server vs client components.** Pages fetch data server-side (no loading spinners, no client-side Supabase queries leaking into the bundle). Client components — `TasksClient`, `VendorsClient`, `BudgetClient`, `ChatInterface` — handle mutations and real-time interaction only.

**AI context injection.** The AI is not agentic. It does not call tools or query the database itself. `lib/wedding-context.ts` serialises the current state of tasks, vendors, budget, and events into a plain object that gets JSON-stringified into the system prompt on every request. Simple, predictable, cheap.

**Nudge caching.** The `/api/nudges` route checks the `ai_nudges` table for a row generated within the last 6 hours before calling Gemini. This means opening the dashboard does not trigger an API call on every visit. You can force a refresh with `/api/nudges?refresh=1`.

**RLS.** Every table has Row Level Security enabled. A helper function `get_user_wedding_id()` returns the caller's `wedding_id` from `wedding_members`, and all policies are written against that. Neither user can read or write rows belonging to a different wedding.

## Database schema

Six tables:

- `weddings` — one row, the wedding record itself
- `wedding_members` — joins users to the wedding, stores role (`owner` / `editor`)
- `events` — the three events with dates and optional venue
- `tasks` — per-event tasks with status, category, assignee, due date
- `vendors` — vendor records with category, status, contact info, quote
- `budget_items` — spend line items linked to an event (or general)
- `ai_nudges` — cached Gemini nudge responses with timestamp

The full schema including RLS policies and seed data is in `supabase-schema.sql`. Run the whole file in the Supabase SQL editor to set everything up.

## Running locally

```bash
cd wedding-planner
npm install
npm run dev
```

Open `http://localhost:3000`. You will be redirected to `/login`.

Before the app does anything useful you need Supabase and a Gemini key — see `SETUP.md` in the repo root for the full walkthrough.

## Design

White background. One accent color (`#B5886A`, a muted warm terracotta). Two fonts — Cormorant Garamond for display text (the days counter, event names, page titles) and DM Sans for everything functional. No shadows on cards, borders only. No gradients. No rounded-full on anything except status badges.

The goal was something that reads as a deliberate tool rather than a generated UI. Inspired by [weddingclaire.com](https://weddingclaire.com).
